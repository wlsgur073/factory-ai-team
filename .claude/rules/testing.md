# 테스트 규칙

## 테스트 프레임워크

- **프레임워크**: 미설정 (추후 Vitest 또는 Jest 도입 예정)
- **E2E**: 미설정 (추후 Playwright 도입 예정)

## 테스트 구조

- **테스트 파일 위치**: 소스 파일 옆 또는 `__tests__/`
- **파일 명명**: `*.test.ts`, `*.test.tsx`
- **테스트 명명**: `describe('컴포넌트명', () => { it('동작 설명') })`

## 커버리지

- **최소 커버리지 목표**: 추후 설정
- **필수 커버리지 대상**: 비즈니스 로직, 솔루션 레지스트리 유틸 함수
- **커버리지 제외 대상**: UI 컴포넌트 (`components/ui/`), 타입 정의

## 테스트 패턴

- **단위 테스트**: 유틸 함수, 레지스트리 함수
- **컴포넌트 테스트**: 주요 페이지 컴포넌트 렌더링 확인
- **E2E 테스트**: 사용자 네비게이션 흐름

### 테스트 작성 원칙

- 각 테스트는 독립적으로 실행 가능해야 합니다
- 테스트 데이터는 테스트 내에서 설정하고 정리합니다
- 외부 의존성은 적절히 모킹합니다
- 테스트 설명은 명확하고 구체적으로 작성합니다
